# ClipBridge Web

这是第 2 步“认证与设备基础”的最小 Web 端壳子。

当前范围只覆盖：

- 登录
- Refresh Token 自动刷新
- 退出登录
- 当前账号信息
- 设备列表

这样做的目标不是把 Web 一次做全，而是先用一个真实浏览器客户端把认证链路跑通。

## 当前目录

```text
web/
├─ assets/         Web 当前直接使用的图标和插画
├─ index.html      页面入口
├─ styles.css      样式与响应式布局
├─ src/
│  ├─ main.js      应用入口，负责启动模块
│  ├─ config/      路由、导航和存储常量
│  ├─ render/      布局渲染、页面片段、图标
│  ├─ services/    接口请求和 token 刷新
│  ├─ state/       页面状态与本地存储
│  └─ utils/       浏览器与格式化工具
└─ README.md
```

## 本地启动

这是一个纯静态目录，不依赖 Node 和打包器。

如果你在仓库根目录下，可以直接启动一个静态文件服务：

```bash
python3 -m http.server 4173 -d web
```

然后打开：

```text
http://127.0.0.1:4173
```

默认后端地址会填：

```text
http://127.0.0.1:18080
```

## 页面说明

- `#/login`
  - 登录页
- `#/dashboard`
  - 当前账号摘要
  - 当前设备摘要
  - 后续页面入口占位
- `#/devices`
  - 设备列表
  - 当前设备标记
- `#/settings`
  - 会话信息查看
  - 退出登录

## 当前行为规则

- 当前默认关闭公开注册
- 如需重新开放注册，需要把后端 `AUTH_ALLOW_REGISTRATION` 改成 `true` 并重启服务
- 受保护页面会先校验本地登录态
- `GET /v1/account/me` 返回 `401` 时，会优先尝试 `POST /v1/auth/refresh`
- refresh 成功后会自动重试原请求
- refresh 失败后会清空本地会话并回登录页

## 提示与反馈规范

- 登录成功
  - 不弹提示，直接进入后续页面
- 刷新设备列表成功
  - 不弹提示，直接刷新列表内容
- 退出登录、设备名称修改成功、强制下线成功
  - 使用短时 `toast`
  - 默认约 `3.2` 秒后自动消失
- 登录失效、页面初始化失败、接口调用失败
  - 使用页面内错误提示
  - 不自动消失，等待用户后续操作或页面切换
- 危险操作
  - 退出登录、强制下线设备保留确认弹窗
- 进行中状态
  - 只通过按钮文案和禁用态表达
  - 不额外弹成功提示或加载提示
- 设备编辑面板
  - 不再单独显示局部反馈
  - 成功统一走 `toast`
  - 失败统一走页面级错误提示
