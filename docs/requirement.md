# ClipBridge 需求说明

## 1. 项目背景

ClipBridge 是对旧项目 `D:\MyProject\cloud-clipboard-sync-old` 的重建版本。

本项目的目标是：

- 功能上尽量覆盖旧版已有能力
- 实现上明显比旧版更清晰、更容易理解
- 结构上更适合作为课程作业分阶段展示

开发优先级说明：

- 第一优先级：后端、Android、Web 联合开发
- 第二优先级：Windows
- 第三优先级：HarmonyOS

## 2. 总目标

本项目最终要实现一个“多端云剪切板系统”，支持同一账号下多设备之间同步文本、文件和分享内容，并提供管理后台与管理员能力。

最终成品需要满足以下要求：

- 支持账号注册、登录、退出登录
- 支持多设备接入与设备管理
- 支持文本剪切板同步
- 支持剪切板历史记录
- 支持文件中转
- 支持文本/文件分享
- 支持公开取件页
- 支持基础管理员后台
- 支持 Android、Web、HarmonyOS、Windows 多端使用

## 3. 旧项目参考范围

本项目需要参考旧项目中的以下模块：

- `backend/`
- `mobile/android/`
- `mobile/harmonyos/`
- `desktop/windows/`
- `web/`

参考重点不是“原样照搬代码”，而是：

- 参考功能边界是否完整
- 参考接口是否够用
- 参考页面与模块划分思路
- 在新项目中用更简单、更清晰的方式重写

## 4. 功能需求

### 4.1 账号与认证

- 用户注册
- 用户登录
- Access Token 与 Refresh Token
- 退出登录
- 修改密码
- 查看当前账号信息

### 4.2 设备管理

- 查看当前账号下的设备列表
- 单设备下线
- 一键下线全部其他设备
- 记录设备平台、设备名、最近在线时间

### 4.3 剪切板同步

- 上传新的剪切板内容
- 支持文本内容同步
- 支持历史记录分页查询
- 支持删除单条历史
- 支持离线补拉
- 支持 ACK 同步游标
- 支持实时推送
- 支持避免设备之间重复回环同步

### 4.4 文件中转

- 文件上传
- 文件列表查询
- 文件下载
- 文件删除
- 文件重命名
- 记录文件大小、类型、哈希、来源设备

### 4.5 分享能力

- 创建文本分享
- 创建文件分享
- 分享列表查询
- 分享提前撤销
- 公开取件页
- 支持过期时间
- 支持阅后即焚
- 支持是否允许复制文本
- 支持加密分享

### 4.6 用户申请能力

- 申请提升存储配额
- 申请提升上传/下载带宽
- 申请管理员权限
- 查询自己的申请记录

### 4.7 管理员能力

- 查看与修改系统设置
- 用户列表查询
- 更新用户状态或额度
- 删除用户
- 审批配额申请
- 审批带宽申请
- 审批管理员申请

### 4.8 各端功能目标

#### 后端

- 提供统一 REST API
- 提供 WebSocket 实时推送
- 提供数据库落库与同步游标管理
- 提供文件存储与分享能力
- 提供管理员相关接口

#### Android

- 登录与退出登录
- 后台同步服务
- 文本剪切板监听与同步
- 历史记录查看
- 文件上传入口
- 系统分享上传
- 设备管理
- 账号设置

#### Web

- 定位为“管理端”而不是系统级剪切板接管端
- 提供 Dashboard、历史记录、文件中心、设备管理、分享管理、申请记录、管理员页面
- 复用后端统一接口

#### HarmonyOS

- 放在最后实现
- 保持与 Android 尽量一致的交互语义
- 支持登录、同步、历史、设备管理、文件中转

#### Windows

- 在后端、Android、Web 基本稳定后接入
- 支持登录、同步中心、历史记录、文件中转、设备管理、账号设置
- 支持托盘常驻与后续安装包打包

### 4.9 AI 附加功能

AI 功能属于附加功能，不影响本项目核心主线。
即使 AI 功能暂时未完成，剪切板同步、历史记录、文件中转、分享和管理后台这些核心能力仍然必须完整可用。

计划引入的 AI 功能如下：

- AI 文本整理
  - 对剪切板文本做摘要、润色、改写、翻译、提取要点、提取待办
- OCR 图转文
  - 对截图、剪切板图片、上传图片执行文字提取
  - 提取后的文字可以继续进入 AI 文本整理流程
- 敏感信息提醒
  - 在同步或分享前检测密码、验证码、手机号、邮箱、API Key 等敏感内容
  - 对用户进行继续/取消提醒
- 智能标题与标签
  - 为历史记录自动生成标题、分类和标签
  - 便于后续在历史页中查找和整理
- 语义搜索
  - 支持用自然语言搜索历史记录
  - 不只依赖关键词匹配

AI 功能建议优先级如下：

1. AI 文本整理
2. 敏感信息提醒
3. OCR 图转文
4. 智能标题与标签
5. 语义搜索

## 5. 非功能要求

- 代码必须可读性强
- 关键逻辑必须保留清晰注释
- 模块划分尽量简单，避免一开始过度抽象
- 接口命名尽量稳定，便于多个客户端共用
- 出错信息尽量直观，便于调试
- AI 功能必须通过后端统一转发，不让客户端直接持有模型密钥
- AI 功能异常时不能阻塞核心同步、分享和历史记录功能

## 6. 部署与环境要求

### 6.1 服务器

- 继续使用 `us2`

### 6.2 旧系统保护要求

- 旧网页地址 `https://hy-us2.xushuangbo.top:18443/dashboard.html` 不能受影响
- 旧服务目录、旧 nginx 配置、旧后端进程都不能直接覆盖
- 新项目必须使用独立部署路径
- 新项目必须使用独立服务名
- 新项目必须使用独立域名

### 6.3 新项目网页端建议

为了尽量不影响旧系统，当前建议新网页端使用：

- `https://clipbridge-us2.xushuangbo.top:18444`

这里的 `18444` 是当前文档给出的推荐新端口，目的是与旧项目的 `18443` 做隔离。
